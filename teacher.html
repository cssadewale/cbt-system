⚙️ Supabase Setup Guide

Run all these SQL statements once in Supabase Dashboard → SQL Editor. This sets up data isolation, security, and all required columns. Run them in order.

🔐 Step 1 — Enable Row Level Security (RLS) on Both Tables

This is the master switch. Without this, all teachers see each other's data.

-- Enable RLS on both tables ALTER TABLE exams ENABLE ROW LEVEL SECURITY; ALTER TABLE results ENABLE ROW LEVEL SECURITY;
🔒 Step 2 — Exams: Teachers Only See & Edit Their Own

Each teacher is completely isolated. They can only read, create, update, and delete their own exams.

-- Teachers can only SELECT their own exams (authenticated) CREATE POLICY "Teachers select own exams" ON exams FOR SELECT TO authenticated USING (auth.uid() = teacher_id); -- Students (anonymous) can read any exam to load it by code CREATE POLICY "Students can read exams by code" ON exams FOR SELECT TO anon USING (true); -- Teachers can only INSERT exams they own CREATE POLICY "Teachers insert own exams" ON exams FOR INSERT WITH CHECK (auth.uid() = teacher_id); -- Teachers can only UPDATE their own exams CREATE POLICY "Teachers update own exams" ON exams FOR UPDATE USING (auth.uid() = teacher_id); -- Teachers can only DELETE their own exams CREATE POLICY "Teachers delete own exams" ON exams FOR DELETE USING (auth.uid() = teacher_id);
🔒 Step 3 — Results: Full RLS Policies (FIXED — Run This Version)

⚠️ The standard subquery approach (SELECT teacher_id FROM exams WHERE id = exam_id) causes a recursive RLS deadlock because the exams table also has RLS enabled. This version uses a SECURITY DEFINER helper function that bypasses RLS safely for the lookup only.

-- Step 3a: Drop old broken policies if they exist DROP POLICY IF EXISTS "Teachers select own results" ON results; DROP POLICY IF EXISTS "Teachers update own results" ON results; DROP POLICY IF EXISTS "Teachers delete own results" ON results; DROP POLICY IF EXISTS "Students can submit results" ON results; -- Step 3b: Create a SECURITY DEFINER helper function. -- This function runs as the DB owner and can read exams without RLS blocking it. CREATE OR REPLACE FUNCTION get_exam_teacher_id(p_exam_id uuid) RETURNS uuid LANGUAGE sql SECURITY DEFINER STABLE AS $$ SELECT teacher_id FROM exams WHERE id = p_exam_id LIMIT 1; $$; -- Step 3c: Recreate policies using the helper function (no recursive RLS) CREATE POLICY "Teachers select own results" ON results FOR SELECT TO authenticated USING (auth.uid() = get_exam_teacher_id(exam_id)); CREATE POLICY "Students can submit results" ON results FOR INSERT TO anon, authenticated WITH CHECK (true); CREATE POLICY "Teachers update own results" ON results FOR UPDATE TO authenticated USING (auth.uid() = get_exam_teacher_id(exam_id)); CREATE POLICY "Teachers delete own results" ON results FOR DELETE TO authenticated USING (auth.uid() = get_exam_teacher_id(exam_id));
📦 Step 4 — Add Anti-Cheat & Answer Columns

Adds columns for per-question answer data, violation counts, and integrity logs.

ALTER TABLE results ADD COLUMN IF NOT EXISTS answers_data JSONB; ALTER TABLE results ADD COLUMN IF NOT EXISTS violations integer DEFAULT 0; ALTER TABLE results ADD COLUMN IF NOT EXISTS violation_log JSONB;

📦 Step 5 — Add Student Mode Columns to Existing Tables

These columns enable registered-student exams and track how each student sat the exam.

-- Add exam_mode column to exams table ALTER TABLE exams ADD COLUMN IF NOT EXISTS exam_mode text DEFAULT 'open'; -- Add student identity columns to results table ALTER TABLE results ADD COLUMN IF NOT EXISTS student_id_ref text; ALTER TABLE results ADD COLUMN IF NOT EXISTS student_type text DEFAULT 'open'; ALTER TABLE results ADD COLUMN IF NOT EXISTS answers_data JSONB; ALTER TABLE results ADD COLUMN IF NOT EXISTS violations integer DEFAULT 0; ALTER TABLE results ADD COLUMN IF NOT EXISTS violation_log JSONB;
👨‍🎓 Step 6 — Create Students Table

This table stores your class roster for registered-mode exams. Each teacher only sees their own students.

CREATE TABLE IF NOT EXISTS students ( id uuid PRIMARY KEY DEFAULT gen_random_uuid(), teacher_id uuid REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL, full_name text NOT NULL, student_id text NOT NULL, class text NOT NULL, created_at timestamptz DEFAULT now(), UNIQUE (teacher_id, student_id) ); -- Enable RLS — teachers only see their own students ALTER TABLE students ENABLE ROW LEVEL SECURITY; CREATE POLICY "Teachers select own students" ON students FOR SELECT USING (auth.uid() = teacher_id); CREATE POLICY "Teachers insert own students" ON students FOR INSERT WITH CHECK (auth.uid() = teacher_id); CREATE POLICY "Teachers delete own students" ON students FOR DELETE USING (auth.uid() = teacher_id); -- Students (anonymous) can read roster to verify their ID during exam CREATE POLICY "Anyone can verify student ID" ON students FOR SELECT USING (true);
⚠️ Step 7 — Fix Existing Teachers Already on the Platform

If teachers registered BEFORE the profiles table was created, they have no profile row and will be blocked on login. Run this SQL to create active profiles for all existing auth users. Change 'buildingmyictcareer@gmail.com' to your actual admin email.

-- Insert active profiles for all existing auth users -- (Run this ONCE after creating the profiles table) -- It skips anyone who already has a profile row INSERT INTO profiles (id, email, full_name, role, status) SELECT u.id, u.email, COALESCE(u.raw_user_meta_data->>'display_name', split_part(u.email,'@',1)), CASE WHEN u.email = 'buildingmyictcareer@gmail.com' THEN 'admin' ELSE 'teacher' END, CASE WHEN u.email = 'buildingmyictcareer@gmail.com' THEN 'active' ELSE 'active' END FROM auth.users u WHERE NOT EXISTS (SELECT 1 FROM profiles p WHERE p.id = u.id) ON CONFLICT (id) DO NOTHING;

After running this, any new teacher who signs up will automatically get a 'pending' profile via the trigger (Step 4 in the admin SQL guide), and the admin must approve them before they can log in.

📷 Step 8 — Add Proctoring Data Column

Required for storing student identity photos and periodic exam snapshots from the anti-cheat system.

ALTER TABLE results ADD COLUMN IF NOT EXISTS proctor_data JSONB; ALTER TABLE results ADD COLUMN IF NOT EXISTS time_taken integer DEFAULT 0; ALTER TABLE results ADD COLUMN IF NOT EXISTS attempt_number integer DEFAULT 1;
👑 Step 9 — Admin RPC Functions (Run in Supabase SQL Editor)

These functions let the admin panel see ALL teachers' data across RLS boundaries. The DROP … CASCADE lines remove any old version first — this fixes the "cannot change return type" error.

-- ══════════════════════════════════════════════ -- Drop old versions first (fixes "cannot change -- return type" error from previous runs) -- ══════════════════════════════════════════════ DROP FUNCTION IF EXISTS admin_get_all_profiles() CASCADE; DROP FUNCTION IF EXISTS admin_get_all_exams() CASCADE; DROP FUNCTION IF EXISTS admin_get_all_results() CASCADE; DROP FUNCTION IF EXISTS admin_set_profile_status(uuid,text) CASCADE; DROP FUNCTION IF EXISTS admin_delete_profile(uuid) CASCADE; -- ══════════════════════════════════════════════ -- admin sees all profiles -- ══════════════════════════════════════════════ CREATE FUNCTION admin_get_all_profiles() RETURNS SETOF profiles LANGUAGE sql SECURITY DEFINER STABLE AS $$ SELECT * FROM profiles ORDER BY created_at DESC; $$; -- ══════════════════════════════════════════════ -- admin sees all exams -- ══════════════════════════════════════════════ CREATE FUNCTION admin_get_all_exams() RETURNS SETOF exams LANGUAGE sql SECURITY DEFINER STABLE AS $$ SELECT * FROM exams ORDER BY created_at DESC; $$; -- ══════════════════════════════════════════════ -- admin sees all results (joined with exam info) -- Uses only columns guaranteed to exist. -- Adjust the column list if your table differs. -- ══════════════════════════════════════════════ CREATE FUNCTION admin_get_all_results() RETURNS TABLE ( id uuid, exam_id uuid, student_name text, student_class text, student_id_ref text, student_type text, score integer, total integer, attempt_number integer, time_taken integer, answers_data jsonb, violations integer, violation_log jsonb, proctor_data jsonb, created_at timestamptz, exams jsonb ) LANGUAGE sql SECURITY DEFINER STABLE AS $$ SELECT r.id, r.exam_id, r.student_name, r.student_class, r.student_id_ref, r.student_type, r.score, r.total, COALESCE(r.attempt_number, 1), COALESCE(r.time_taken, 0), r.answers_data, COALESCE(r.violations, 0), r.violation_log, r.proctor_data, r.created_at, jsonb_build_object( 'subject', e.subject, 'teacher_id', e.teacher_id, 'csv_data', e.csv_data ) AS exams FROM results r LEFT JOIN exams e ON e.id = r.exam_id ORDER BY r.created_at DESC; $$; -- ══════════════════════════════════════════════ -- admin can approve / reject teachers -- ══════════════════════════════════════════════ CREATE FUNCTION admin_set_profile_status(p_id uuid, p_status text) RETURNS void LANGUAGE sql SECURITY DEFINER AS $$ UPDATE profiles SET status = p_status WHERE id = p_id; $$; CREATE FUNCTION admin_delete_profile(p_id uuid) RETURNS void LANGUAGE sql SECURITY DEFINER AS $$ DELETE FROM profiles WHERE id = p_id; $$;

⚠️ If you still get a column error, run Step 8 first to add the proctor_data, time_taken, and attempt_number columns, then re-run Step 9.

🔬 Step 10 — Verify Your Setup (Run to Test)

Run this in the Supabase SQL Editor to confirm everything is working. You should see rows returned for exams and results if data exists.

-- Test 1: Confirm RLS policies exist on results table SELECT policyname, cmd, roles FROM pg_policies WHERE tablename = 'results'; -- Test 2: Confirm the helper function exists SELECT proname FROM pg_proc WHERE proname = 'get_exam_teacher_id'; -- Test 3: Confirm admin RPC functions exist SELECT proname FROM pg_proc WHERE proname IN ( 'admin_get_all_results', 'admin_get_all_profiles', 'admin_get_all_exams' ); -- Test 4: Count all results (runs as superuser, bypasses RLS) SELECT COUNT(*) AS total_results FROM results; SELECT COUNT(*) AS total_exams FROM exams;
