⚙️ Supabase Setup Guide

Run all these SQL statements once in Supabase Dashboard → SQL Editor. This sets up data isolation, security, and all required columns. Run them in order.

🔐 Step 1 — Enable Row Level Security (RLS) on Both Tables

This is the master switch. Without this, all teachers see each other's data.

-- Enable RLS on both tables ALTER TABLE exams ENABLE ROW LEVEL SECURITY; ALTER TABLE results ENABLE ROW LEVEL SECURITY;
🔒 Step 2 — Exams: Teachers Only See & Edit Their Own

Each teacher is completely isolated. They can only read, create, update, and delete their own exams.

-- Teachers can only SELECT their own exams CREATE POLICY "Teachers select own exams" ON exams FOR SELECT USING (auth.uid() = teacher_id); -- Teachers can only INSERT exams they own CREATE POLICY "Teachers insert own exams" ON exams FOR INSERT WITH CHECK (auth.uid() = teacher_id); -- Teachers can only UPDATE their own exams CREATE POLICY "Teachers update own exams" ON exams FOR UPDATE USING (auth.uid() = teacher_id); -- Teachers can only DELETE their own exams CREATE POLICY "Teachers delete own exams" ON exams FOR DELETE USING (auth.uid() = teacher_id);
🔒 Step 3 — Results: Teachers Only See Results From Their Own Exams

A result belongs to a teacher if it belongs to an exam that teacher created. Students (anonymous) can still submit results.

-- Teachers can SELECT results only from their own exams CREATE POLICY "Teachers select own results" ON results FOR SELECT USING ( auth.uid() = (SELECT teacher_id FROM exams WHERE id = exam_id) ); -- Anyone (students) can INSERT results (no auth required for submission) CREATE POLICY "Students can submit results" ON results FOR INSERT WITH CHECK (true); -- Teachers can UPDATE results from their own exams CREATE POLICY "Teachers update own results" ON results FOR UPDATE USING ( auth.uid() = (SELECT teacher_id FROM exams WHERE id = exam_id) ); -- Teachers can DELETE results from their own exams CREATE POLICY "Teachers delete own results" ON results FOR DELETE USING ( auth.uid() = (SELECT teacher_id FROM exams WHERE id = exam_id) );
📦 Step 4 — Add Anti-Cheat & Answer Columns

Adds columns for per-question answer data, violation counts, and integrity logs.

ALTER TABLE results ADD COLUMN IF NOT EXISTS answers_data JSONB; ALTER TABLE results ADD COLUMN IF NOT EXISTS violations integer DEFAULT 0; ALTER TABLE results ADD COLUMN IF NOT EXISTS violation_log JSONB;
🔗 Step 5 — Your Student Portal Link

Share this with your students. Each exam appends ?code=XXXXXX automatically.
